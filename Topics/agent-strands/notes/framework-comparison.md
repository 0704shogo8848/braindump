# Strands Agents と他フレームワークの比較

## 比較対象

| 観点 | Strands Agents | LangGraph | CrewAI | AutoGen |
|------|---------------|-----------|--------|---------|
| **設計思想** | モデル駆動 | グラフベース | ロール/チーム | マルチエージェント会話 |
| **学習曲線** | 非常に低い | 高い | 低い | 中程度 |
| **コード量** | 最小（3行〜） | 多い | 中程度 | 中程度 |
| **本番実績** | AWS 内部で広く使用 | 多企業で導入済み | エンタープライズ機能あり | 研究寄り |
| **MCP 統合** | ネイティブ | プラグイン | 限定的 | 限定的 |
| **クラウド統合** | AWS に深い | クラウド非依存 | クラウド非依存 | Azure 寄り |

## 詳細比較

### vs LangGraph
- **LangGraph**: LangChain エコシステムの拡張。ワークフローをグラフ（ノード＋エッジ）で構造化。状態管理が明示的。永続ワークフローをサポート
- **Strands の優位点**: API がシンプル、セットアップが速い、MCP ネイティブ
- **LangGraph の優位点**: ワークフロー制御が細かい、エコシステムが大きい、クラウド非依存

### vs CrewAI
- **CrewAI**: 「クルー」概念でエージェントチームを構成。ロールとゴールが直感的
- **Strands の優位点**: より低レベルの柔軟性、AWS 統合、MCP サポート
- **CrewAI の優位点**: エンタープライズコントロールプレーン、ローコードツール、有料サポート

### vs AutoGen（Microsoft）
- **AutoGen**: マルチエージェント会話に特化。研究・プロトタイピング向け
- **Strands の優位点**: 本番運用への道筋が明確、デプロイオプションが豊富
- **AutoGen の優位点**: 研究の柔軟性、Microsoft エコシステムとの親和性

## 使い分けの指針

- **Strands**: AWS 上で迅速に開始したい、MCP エコシステムを活用したい、シンプルな API を好む
- **LangGraph**: LangChain 既存ユーザー、細かいワークフロー制御が必要、耐久性のあるワークフロー
- **CrewAI**: エージェントチームの迅速な構築、ローコードツール、エンタープライズサポートが必要
- **AutoGen**: プロトタイピング・研究、マルチエージェント会話の柔軟性が必要
