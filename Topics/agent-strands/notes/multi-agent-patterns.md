# Strands Agents マルチエージェントパターン

## 概要

Strands 1.0 で導入された4つのマルチエージェントプリミティブにより、単一エージェントの model/tool/prompt の組み合わせを自然に拡張してマルチエージェントシステムを構築できる。

## 4つのコアプリミティブ

### 1. Agents-as-Tools（ツールとしてのエージェント）
専門エージェントをツールに変換し、オーケストレーターエージェントが動的にドメインエキスパートに相談できる。階層的委任パターン。制御はオーケストレーターが保持。

### 2. Swarm（スウォーム）
アリのコロニーや鳥の群れに着想を得た自律エージェントの集合体。個々のエージェントが自律的に動作しつつ、集合知として個々の能力を超える成果を生む。

### 3. Graph（グラフ）
明示的なワークフローを条件分岐・決定ポイント付きで定義。承認フロー・品質ゲート等、特定の手順を必要とするプロセスに適する。ビジネスルールに沿った正しい順序の実行を保証。

### 4. 組み合わせ
- Swarm の中に Graph を含められる
- Graph が Swarm をオーケストレーションできる
- どのパターンも Agents-as-Tools を利用可能
- 段階的に採用可能: 単一エージェント → ツールとしての専門家追加 → Swarm → Graph

## セッション管理

- 各エージェントにユニークIDを付与
- リモートデータストアからのエージェント状態取得
- 同一セッション内の並行エージェント管理
- デプロイ・スケーリング・再起動をまたいだコンテキスト維持
- DAO パターンでストレージバックエンドを拡張可能
- デフォルトバックエンド: ローカルファイルシステム、Amazon S3

## メモリ統合

- **短期メモリ**: AgentCore Memory との連携で会話メッセージを自動同期
- **長期メモリ**: セッション終了時や重要なインタラクションポイントで長期記憶に保存
- **エージェント間コンテキスト共有**: マルチエージェント実行内で他エージェントのコンテキストにアクセス
- **パターン対応セッション管理**: ハンドオフ・Swarm インタラクション・Graph 遷移を理解

## 非同期サポート（v1.0）

v1.0 でイベントループアーキテクチャを改善し、スタック全体で async 操作をサポート。ツールとモデルプロバイダーがブロッキングなしで非同期実行可能。
